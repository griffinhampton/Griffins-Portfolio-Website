FNAD World

One of my most ambitious projects to date, and my intro into managing data.

This project was born from an undying curiosity to venture into CSV file maps, and served as a much needed refresher with excel. It would be no stretch to say that this game prepared me for eventually becoming the coding contractor I am today. What started as a silly game for me and my friends to play in high school turned into the spark that ignited my love for computer science.

Game Development CSV File Formatting Python GUI Development UX/UI Design

Intro

FNAD World was my first major college project and one of the most technically ambitious games I’ve made. Inspired by FNAF World, I set out to expand my own FNAD universe while also pushing myself to explore new programming challenges. The biggest hurdle — and greatest learning opportunity — was figuring out how to use CSV files to auto-generate massive sprite maps. What started as a fun idea for a class project quickly became something much more meaningful: a way to blend my love for coding with practical skills that apply far beyond game development.

Skills Applied

Developing FNAD World required me to step into unfamiliar territory and adapt quickly. I used Python as the core language, but this time I focused heavily on data management. By leveraging CSV parsing, I built a system that could dynamically read, interpret, and generate large maps from raw spreadsheet data — essentially automating what would have otherwise been hours of manual work. Along the way, I reinforced my understanding of Excel data structures, file I/O operations, and modular program design. This project was also a lesson in balancing GUI design and gameplay logic, ensuring the game was both functional and fun to interact with.

The Future of FNAD World

While FNAD World is a finished project, the techniques I developed for it are skills I carry into nearly everything I build. The ability to take structured data (like a CSV file) and transform it into something interactive has obvious applications in software engineering, data processing, and automation. Looking forward, I see this project as a prototype for more advanced tools and systems that merge entertainment with real-world utility. And as with FNAD 2, it reinforced the simple truth that I love coding too much to stop at “good enough.”

Conclusion

FNAD World wasn’t just another game — it was a gateway into understanding how data can shape entire systems. It taught me that even something as simple as a spreadsheet can unlock creativity when paired with the right logic and design. Most importantly, it reminded me why I code: because I love solving problems, building experiences, and pushing myself to create things that didn’t exist before.
